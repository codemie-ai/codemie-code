## Summary

When a codemie-claude session closes/exits, codemie runs the equivalent of
`codemie analytics --report --session <session_id> --report-format json --report-output ./docs/codemie/analytics/codemie-analytics-{session_id}.json`
programmatically as part of the finalization step — not as a separately-invoked CLI command.
Enabled by default; a codemie-claude flag disables it. The same behavior applies to
codemie-codex and codemie-opencode.

## Acceptance Criteria

- On codemie-claude session exit, the analytics report generation runs automatically as part of finalization.
- The report is generated by calling the `analytics --report` logic programmatically (in-process), not by spawning a separate `codemie` CLI process.
- Report parameters: session = the just-finished session id; format = json; output path = `./docs/codemie/analytics/codemie-analytics-<session_id>.json`.
- The feature is enabled by default.
- A codemie-claude flag disables the feature.
- codemie-codex and codemie-opencode exhibit the same behavior (auto-report on exit, default-on, disable flag).
- Failure to generate the report must not break/abort the session finalization.
